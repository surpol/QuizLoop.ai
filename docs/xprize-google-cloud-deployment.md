# XPRIZE Google Cloud Deployment Path

This document describes the production path for the Build with Gemini XPRIZE version of QuizLoop.ai.

## Goal

The XPRIZE submission should prove that QuizLoop is a business operating with AI. The deployed system should not depend on a MacBook, ngrok tunnel, or local-only Ollama server.

## Recommended Stack

```text
Browser / PWA
  -> Cloud Run web service
  -> SQLite-compatible local store for prototype evidence
  -> Gemini API for learner reports and business intelligence
  -> Cloud Logging for model/product execution logs
  -> Cloud Storage for exported evidence packets
```

For the first business pilot, keep the product simple:

- Cloud Run hosts the Node web server.
- Gemini API generates learner progress reports from saved quiz evidence.
- `/api/evidence` exports product evidence for the submission.
- Cloud Logging captures request and model-run logs.
- Cloud Storage stores weekly evidence snapshots and screenshots.

## Environment

```text
HOST=0.0.0.0
PORT=8080
ACCORDIAN_DB_PATH=/data/quizloop.sqlite
GEMMA_BASE_URL=https://your-public-gemma-host.example
GEMMA_MODEL=gemma4:e2b
GEMINI_API_KEY=your-gemini-api-key
GEMINI_MODEL=gemini-2.5-flash
```

## Required Product Proof

The deployed app should expose:

```text
GET  /api/health
GET  /api/evidence
POST /api/notes/:noteId/learner-report
```

`/api/health` proves the app is deployed.

`/api/evidence` proves the business has activity: notes, quizzes, attempts, user actions, and model runs.

`/api/notes/:noteId/learner-report` proves Gemini is part of the deployed product, not only the writeup.

## Future Durable Database

The local SQLite file is fine for prototype evidence, but a real business should migrate hosted data to one of:

- Cloud SQL for a traditional relational backend.
- Firestore for user, note, quiz, and report documents.
- Turso/LibSQL if we want to keep the SQLite mental model.

The iOS product can remain local-first. The cloud business layer is for reports, pilots, educator dashboards, and competition evidence.

## Submission Evidence Checklist

- Cloud Run service URL.
- Screenshot of Cloud Run deployment.
- Screenshot of Gemini API usage.
- Sample `/api/evidence` JSON.
- Sample learner report generated by Gemini.
- User count, quiz count, notes count, and repeated quiz-loop count.
- Cost notes from Google Cloud billing.
- Short explanation of how Gemma and Gemini play different roles.
