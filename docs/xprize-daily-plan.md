# XPRIZE Daily Execution Plan

QuizLoop.ai is web-only for XPRIZE. The daily goal is simple: improve the product, talk to users, and collect evidence.

Deadline: August 17, 2026

## North Star

Build an AI-operated AWS certification prep business.

The first customer is a learner preparing for:

- AWS Certified Cloud Practitioner
- AWS Certified Solutions Architect - Associate

## Daily Loop

Do these five things every day:

1. Ship one product improvement.
2. Talk to one potential user.
3. Capture one piece of evidence.
4. Run one quiz-loop test.
5. Write one insight in the evidence log.

## Week 1: Launch The Wedge

- Publish AWS cert landing page.
- Add pilot signup capture.
- Add Certs source type in the app.
- Add AWS Cloud Practitioner and Solutions Architect templates.
- Create a simple user outreach script.
- Ask 10 people if they are studying for or interested in cloud certification.
- Record every response.

## Week 2: Make The Product Useful

- Improve AWS quiz quality.
- Make scenario/tradeoff questions stronger.
- Add no-duplicate enforcement tests for certification notes.
- Add Gemini learner report UI.
- Add evidence dashboard view.
- Create one sample AWS Cloud Practitioner study note.
- Take 5 quiz loops and document how the next quiz changes.

## Week 3: Get Pilot Users

- Recruit 5 pilot users.
- Watch 2 users use the app live.
- Add onboarding copy based on where they get confused.
- Add feedback form after first quiz.
- Add Stripe/payment link or manual paid pilot invoice.
- Ask every pilot user whether they would pay.

## Week 4: Revenue Attempt

- Convert at least 1 pilot user into a paid user.
- Create a $9/month or $29 one-time pilot offer.
- Save revenue evidence, even if tiny.
- Track expenses.
- Write a weekly business evidence summary.

## Month 2: Prove Retention

- Get users to complete more than one quiz loop.
- Add weekly Gemini progress reports.
- Add email/report export.
- Add teacher/tutor summary mode.
- Track repeat usage.
- Improve based on failed or boring quiz loops.

## Month 3: Package The Story

- Tighten product demo.
- Build final evidence dashboard.
- Export usage, revenue, cost, and model-run evidence.
- Write 500-1000 word narrative.
- Record 3-minute demo.
- Prepare public repo and deployment instructions.
- Submit early.

## Evidence We Need

- Leads captured.
- Active users.
- Notes created.
- Quizzes completed.
- Quiz score movement.
- Gemini report runs.
- User feedback.
- Revenue evidence.
- Expense evidence.
- Product logs.
- Screenshots of Google Cloud deployment and Gemini usage.
